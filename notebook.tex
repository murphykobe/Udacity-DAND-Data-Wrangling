\subsection{Data Wrangling Report}\label{data-wrangling-report}}

    In this document I'll go through the data wrangling process for my
project This process is consist of three part: - Gathering - Assessing -
Cleaning

    \hypertarget{gathering}{%
\subsubsection{Gathering}\label{gathering}}

    There's three different data source for this project. - First for the
archive tweet data, we just using an browser dowloading the csv file -
Second, for the image prediciton, we are using requests package
downloading programmatically with python, save it into the workspace -
At last, the tweet retweet\# and favorite\# data we are using tweeter
API and tweepy package to query each tweet data and saved into a JSON
file.

    The main chanllenge here I met is to gathering twitter data since it's
running in the backgroud it's not easy to check the status, after I
added exception handle and wait rate limit parameter it finally gathered
all the data into the list I needed.

    After all the data is been stored locally, it's being imported into
jupyter notebook as dataframe.

    \hypertarget{assessing}{%
\subsubsection{Assessing}\label{assessing}}

    \hypertarget{quality}{%
\subsubsection{Quality}\label{quality}}

\textbf{tweet archive table} - Missing value in expanded\_urls
-\textgreater{} Using existing tweet\_id recosntruct urls - Duplicate \&
error values in expanded\_urls, repeated urls in one data entry
-\textgreater{} same above - There's error in dog names, example like
`a' or `an' is not a name, also a lot of entry is None -\textgreater{}
using python string function remove non-capitalized names - 0 value on
rating\_numerator also there's values that not 10 for
rating\_denominator -\textgreater{} checking original tweet find correct
rating value, remove non-rating tweet - Duplicate on dog types, there's
serval dog have multiple properties which is incorrect -\textgreater{}
checking original tweet text to find correct dog type - We only want
original ratings (no retweets) that have images -\textgreater{} remove
all the column have retweet data

\textbf{predition table} - Duplicate tweet\_id column header
-\textgreater{} join with tweet archive table - 2075 total entry,
there's 200+ missing data entry compare to tweet archive table
-\textgreater{} we will using inner join to exclude all the missing data
- Dog type name convention is not the same (p1,p2,p3), there's lowercase
also underscore into it -\textgreater{} using python string function to
replace underscore '\_`with' '

\textbf{count table} - there's duplicate data in this dataset need to be
removed -\textgreater{} using df.drop\_duplicates() function

    \hypertarget{tidiness}{%
\subsubsection{Tidiness}\label{tidiness}}

\begin{itemize}
\item
  count table and prediction table can be inlcuded in the tweet archive
  table -\textgreater{} join all three table together
\item
  the dog types (doggo floofer pupper puppo) can be moved into 1 single
  column since every dog only have one of these properties
  -\textgreater{} melt these column into one column
\item
  Timestamp datatype and also other column datatype need to be changed
  -\textgreater{} changing column datatype to correct one using
  df.astype()
\item
  source in tweet archive table can be shortened iPhone,Vine,Twitter
  Web,TwitterDeck, help shorten the width of the column -\textgreater{}
  shorten the column with better column values
\item
  There's decimal in img\_num column which should be integer
  -\textgreater{} changing column datatype to correct one using
  df.astype()
\end{itemize}

    \hypertarget{cleaning}{%
\subsubsection{Cleaning}\label{cleaning}}

    Cleaning is the last step, I used the method mentioned above which is
both programmatically and manual(visually), of course the manual method
have a lot of work and less effcient but sometimes it's a different view
to find \& solve problems that code/functions can't see.

    \hypertarget{conclusion}{%
\subsubsection{Conclusion}\label{conclusion}}

    From this project I found that data wrangling is definitely a must skill
set to doing any data analaysis. Without tidy/clean data we can't get
the best effect or correct result of any analysis or visualization.
However I do feel like this is just a tip of the iceberg and there's
much more to learn and practice. Practice makes perfect, with more
experience I'll have a better chance facing any challenge in the future.
